Cliente-Server

Essa expressão é utilizada no meio da tecnologia para se referir a um modelo de comunicação que vincula vários outros dispositivos informáticos através de uma rede. O cliente, neste contexto, faz solicitações ao servidor, que é responsável por atender a esses requisitos.

A arquitetura do Client-Server é simples, basta saber que as tarefas são distribuídas entre os servidores (que fornecem os serviços) e os clientes (que exigem os serviços).

Linguagem de Programação

É uma linguagem padronizada formada por um conjunto de regras sintáticas e semânticas, que servem de implementação de um código fonte, que pode ser compilado e transformado em um programa de computador ou apenas ser usado como script interpretado, que informará instruções determinadas de tarefas específicas de processamento ao computador.

As linguagens mais utilizadas no mercado são: JavaScript, Python, Java, PHP e C# .

HTML é uma linguagem de programação?

Não, HTML caracteriza-se como um arquivo de marcação, que são delimitados como nomes que o navegador consegue interpretar, cada um destes delimitadores, tem uma função e comportamento específico.

O HTML não pode criar atividades dinâmicas, ao invés disso com o HTML os usuários podem criar e estruturar seções, parágrafos e links utilizando elementos, tags e atributos.

Ferramentas utilizadas

VsCode é um editor de código-fonte desenvolvido pela Microsoft para Windows, Linux e macOS. Ele inclui suporte para depuração, controle de versionamento Git incorporado, realce de sintaxe, complementação inteligente de código, snippets e refatoração de código.

Estrutura básica - Tags

DOCTYPE - deve ser sempre a primeira tag a aparecer no HTML, ela indica ao browser qal versão da linguagem está sendo usada no momento, no caso, o HTML5, na qual a versão do doctype é muito mais simples.

HTML - a abertura e o fechamento desta tag delimita o documento, assim, todas as demais tags devem estar no meio.

HEAD - contém informações que não são transpostas visivelmente para o usuário/leitor do documento. São dados implícitos, de uso e controle do documento: vinculação com outros arquivos, aplicação de lógica de programação de scripts e metadados.

TITLE - define o título da página, aquele que aparece na aba/janela do navegador, essa tag permanece entre a tag head.

BODY - trata-se do documento em si, ou seja, a informação legível para o usuário/leitor do documento. É todo e qualquer texto que se deseja apresentar, assim como toda e qualquer forma de mídia de saída (imagens, sons, miniaplicativos embutidos, conteúdo multimídia, etc). Além disso, toda a apresentação de entrada de dados (formulários) também se aplica neste seção do documento.

Cabeçalhos e Parágrafos

Os cabeçalhos possuem aparência diferenciada do resto do conteúdo do texto, por isso é definido pelas tags h1 até h6, conforme os números diminuem, o tamanho da letra também diminui. h1= maior/mais relevante e h6= menor/menos relevante .

Os parágrafos são gerados por meio da tag "p" esse é um exemplo de tag cuja disposição ma tela se dá em forma de bloco, ou seja, cada vez que um parágrafo é escrito ele põe-se um em baixo do outro.

(Voltar)

Atributos das Tags

Imagens

Feito por meio da tag "img" e recebe o atributo "src" , o endereço da imagem pode ser carregado. Outros dois atributos são o "alt" , que indica um texto alternativo que será exibido na tela caso o arquivo não possa ser carregado e "title" , que indica o texto que aparecerá como tooltip ao passar o mouse sobre a figura.

Links

Normalmente são utilizados para direcionar os usuários para outras páginas, ou para outras partes da mesma página é utilizado a tag "a" e possui o atributo "href" no qual é indicado o endereço de certo link.

Listas

São elementos úteis para organizar e ordenar itens que estão relacionados de alguma forma, é possível criar três tipos de listas as quais são: ordenadas (tag "ol"), não ordenadas (tag "ul") e de definição (por meio da tag dl).

Formatação dos Textos

  • b e strong = negrito/texto forte

  • i e em = texto em itálico/ênfase

  • sub e sup = sobrescrito e subscrito

  • small = para textos menores que o normal

  • mark = para textos destacados
